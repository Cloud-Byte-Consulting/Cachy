# Release Packaging MVP Readiness

## Status

Ready for the current MVP scope.

## Scope

The release packaging MVP covers release-candidate artifact generation and validation:

- Manual and `v*` tag release workflow for the unified `cachy` CLI.
- CGO-free builds for Windows, macOS, and Linux on AMD64 and ARM64.
- Windows ZIP archives and macOS/Linux tar.gz archives.
- SHA-256 checksum sidecars for each platform archive.
- Non-published multi-arch Docker OCI artifact for `linux/amd64` and `linux/arm64`.
- SBOM JSON evidence for source, binary archives, and Docker OCI artifacts where practical.
- Generated release notes with artifacts, verification steps, and known limitations.
- Install smoke coverage for archive extraction, PATH lookup, `cachy doctor`, proxy startup, and
  `/healthz`.

## Supported Behavior

Users can download a platform archive, extract it, place `cachy` on `PATH`, run `cachy doctor`, and
start `cachy proxy` without Python, Rust, Node, or a compiler.

The release workflow builds the unified `cachy` binary for:

```text
windows/amd64
windows/arm64
darwin/amd64
darwin/arm64
linux/amd64
linux/arm64
```

Release artifacts include checksums and SBOM evidence. Docker packaging produces a local OCI archive
for headless/server review without publishing to a registry.

## Known Boundaries

- Public package publishing requires explicit owner approval.
- Registry push, signing, notarization, auto-update, Winget, Scoop, Homebrew, deb, and rpm packaging
  are later work.
- Docker images are built as non-published OCI artifacts in the current workflow.
- The current SBOM evidence is intentionally lightweight JSON generated by Cachy release tooling;
  deeper ecosystem SBOM formats can be added later.
- The install smoke test validates the runner platform directly. Cross-OS install validation can be
  expanded when Windows and macOS runners are added.
